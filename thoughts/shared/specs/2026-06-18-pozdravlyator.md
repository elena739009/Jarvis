# Поздравлятор — Спецификация MVP

_Дата: 2026-06-18_

---

## Executive Summary

«Поздравлятор» — веб-сервис и Telegram-бот для создания персонализированных видеопоздравлений из фото и музыки. Целевая аудитория — обычные люди (не технари), которые хотят поздравить близких красивым видео, но не умеют монтировать. Монетизация: бесплатно с водяным знаком или разовая оплата/подписка для чистого видео.

---

## Проблема

Люди хотят поздравить близких оригинально, но:
- Монтаж видео — сложно и долго
- Canva/iMovie — слишком много шагов
- Готовые шаблоны — безликие и одинаковые

Текущее решение: WhatsApp-фото + голосовое. Это и есть конкурент.

---

## Метрики успеха

| Метрика | Цель (3 мес.) |
|---------|--------------|

## Пользователи

**Основные персоны:**
1. **Мама, 35–45 лет** — хочет поздравить ребёнка, мужа или родителей. Не разбирается в технологиях. Оценивает простоту и эмоцию.
2. **Подросток, 12–18 лет** — делает сюрприз другу или родным. Любит быстрый результат, делится в соцсетях.
3. **Любой человек** — коллеги, друзья, бабушки. Если понял как — уже готов.

**Технический уровень:** минимальный. Интерфейс — как инстаграм, никаких терминов.

## Пользовательский путь

### Сайт (основной)

```
1. Открывает сайт → сразу видит форму создания (без регистрации)
2. Загружает фото (до 30 штук) и/или короткие видео
3. Загружает MP3 или выбирает из готовых треков (MVP)
4. Вводит текст поздравления
5. Выбирает стиль текста: бегущая строка ИЛИ финальная открытка
6. Нажимает «Создать видео»
   - Если лёгкое (только фото, короткий трек) → рендер прямо на странице, прогресс-бар ~1-2 мин
   - Если тяжёлое (видео в составе, длинный трек) → «Отправим в Telegram когда будет готово» (пользователь даёт свой Telegram username)
7. Превью готового видео прямо на сайте
8. Хочет скачать без водяного знака → выбор:
   - Разовая оплата (100–200 ₽) через Telegram Stars или ссылку на оплату
   - Подписка (300 ₽/мес.)
   - Бесплатно — с водяным знаком
9. Скачивает MP4 ← ссылка действует 24 часа
```

### Telegram-бот

```
1. /start → бот объясняет что умеет
2. Пользователь отправляет фото (по одному или альбомом, до 30)
3. Отправляет MP3
4. Вводит текст поздравления
5. Бот спрашивает: бегущая строка или финальная открытка?
6. Бот принимает задачу → «Готовлю видео, подожди ~2 минуты»
7. Отправляет ссылку на превью или само видео (если < 50MB — прямо в чат)
8. Оплата через Telegram Stars прямо в боте
```

## Функциональные требования

### P0 — Must Have (MVP)

- [ ] Загрузка фото (JPEG/PNG, до 30 штук, макс 20MB каждое)
- [ ] Загрузка аудио (MP3, макс 15MB)
- [ ] Ввод текста поздравления (до 500 символов)
- [ ] Выбор стиля текста: бегущая строка / финальная открытка
- [ ] Автоматический расчёт тайминга слайдов под длину трека
- [ ] Рендеринг видео через FFmpeg (MP4, 1080p)
- [ ] Водяной знак «Поздравлятор» на бесплатном видео
- [ ] Превью перед скачиванием
- [ ] Скачивание MP4 (ссылка 24 часа)
- [ ] Оплата через Telegram Stars (снятие водяного знака)
- [ ] Telegram-бот с базовым флоу
- [ ] Async-очередь для тяжёлых задач (Celery + Redis)

### P2 — Nice to Have (будущее)

## Техническая архитектура

### Стек

| Слой | Технология | Обоснование |
|------|-----------|-------------|
| Backend | Python + FastAPI | Лучший выбор для FFmpeg и AI-библиотек. Удобен при вайбкодинге |
| Видеорендеринг | FFmpeg + MoviePy | Бесплатно, полный контроль, стандарт индустрии |
| Очередь | Celery + Redis | Async-обработка тяжёлых видео |
| База данных | PostgreSQL | Пользователи, задачи, платежи |
| Файловое хранилище | Локальный диск → S3 (после 1000 видео/день) | Простота старта |
| Frontend | Next.js / или простой HTML + HTMX | HTMX проще для вайбкодинга |
| Telegram-бот | python-telegram-bot v20 (async) | Нативная поддержка Stars |
| Хостинг | VPS Hetzner / Selectel / Timeweb | 4 CPU, 8GB RAM достаточно для старта |
| CDN | Cloudflare | Бесплатно, ускоряет раздачу видео |

### Модель данных

```
User
  - id, telegram_id, created_at
  - subscription_active, subscription_until
  - stars_balance

Job (задача рендеринга)
  - id, user_id (nullable для анонимов), status
  - photos_path[], audio_path, text, text_style
  - output_path, watermarked_output_path
  - created_at, expires_at (= created_at + 24h)
  - is_paid

Payment
  - id, job_id, user_id
  - amount, currency, provider (stars|yookassa)
  - status, provider_payment_id, created_at
```

### Компоненты системы

```
[Пользователь]
     │
     ├── [Сайт (Next.js/HTMX)]
     │        │
     │        ▼
     │   [FastAPI Backend]
     │        │
     │        ├── [PostgreSQL] ← пользователи, задачи
     │        ├── [Redis] ← очередь задач
     │        └── [Файловое хранилище] ← фото, аудио, видео (24h TTL)
     │                  │
     │             [Celery Worker]
     │                  │
     │             [FFmpeg / MoviePy]
     │                  │ (готово)
     │             [Telegram Bot API] → уведомляет пользователя
     │
     └── [Telegram Bot]
              │
         [FastAPI Backend] (тот же)
```

### FFmpeg: схема сборки видео

```
1. Принять N фото
2. Вычислить duration_per_slide = track_duration / N
3. Для каждого фото → создать видеоклип (ffmpeg -loop 1 -t duration)
4. Concatenate все клипы
5. Наложить аудиодорожку
6. Наложить текст:
   - Бегущая строка: drawtext с x=w-mod(t*200,w+tw) эффектом
   - Финальная открытка: drawtext на последних 3 секундах
7. Наложить водяной знак (если не оплачено)
8. Экспорт в MP4 H.264, 1080p
```

## Безопасность

- Анонимные пользователи: сессия по cookie (session_id), привязывается к Job
- Файлы: хранятся по случайному UUID, не угадать URL
- Загрузка: проверка MIME-типа + магических байт (не только расширение)
- Размер файлов: жёсткие лимиты (фото 20MB, аудио 15MB, видео 30MB)
- Автоудаление файлов через 24 часа (cron-задача каждый час)
- Rate limiting: не более 3 задач в час для анонима, 20 для платящего

## Открытые вопросы для реализации

1. **Хостинг видео-превью**: стриминг через FastAPI или ссылка на файл? Рекомендую: ссылка на файл через Cloudflare для скорости.
2. **HEIC (iPhone-фото)**: нужна библиотека `pillow-heif` для конвертации перед FFmpeg.
3. **Качество рендера vs скорость**: для MVP — preset `fast`, для платных — `slow` (лучше качество).
4. **Telegram Stars курс**: 1 Star ≈ 0.013$ (Telegram устанавливает). При 200₽ нужно ~200 Stars. Проверить актуальный курс при интеграции.
5. **Очистка файлов**: cron каждый час удаляет файлы с `expires_at < now()`.

## Приложение: Рекомендуемый стек для вайбкодинга

Порядок создания с Claude Code:

```
1. FastAPI backend + PostgreSQL (модели + миграции)
2. FFmpeg pipeline (рендер одного видео)
3. Celery + Redis (очередь)
4. Простой HTML-фронт (форма загрузки + прогресс)
5. Telegram-бот (базовый флоу)
6. Telegram Stars оплата
7. Водяной знак
8. Автоудаление файлов
9. Деплой на VPS (Docker Compose)
```

Минимальный VPS для старта: **4 vCPU, 8GB RAM, 80GB SSD** (~15–20$/мес на Hetzner).
