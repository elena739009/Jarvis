# Поздравлятор — План реализации MVP

_Дата: 2026-06-18 | Спек: 2026-06-18-pozdravlyator.md_

---

## Принципы

- Каждый этап = рабочая демо-версия
- Сначала самое ценное: рендеринг видео
- Деплой на VPS только на этапе 7
- Каждый этап можно начать через Claude Code командой ниже

---

## Этап 1 — Ядро рендеринга (2–3 дня)

**Цель:** Из фото + MP3 → MP4 файл на диске.

**Задачи:**
- [ ] Установить FFmpeg и MoviePy
- [ ] Скрипт `render_video.py`: принимает список путей к фото, путь к MP3, текст
- [ ] Автоматический расчёт `duration_per_slide = audio_duration / len(photos)`
- [ ] Режим 1: бегущая строка (`drawtext` с анимацией по X)
- [ ] Режим 2: финальная открытка (текст на последних 3 сек)
- [ ] Наложение водяного знака (SVG/PNG логотип)
- [ ] Экспорт в MP4 1080p H.264
- [ ] Тест: создать 5 разных видео вручную

**Команда для Claude Code:**
```
Создай render_video.py — скрипт на Python, который принимает:
- список путей к JPEG-фото
- путь к MP3
- текст поздравления
- режим текста (scrolling / card)
- флаг watermark (bool)
И собирает MP4 через MoviePy + FFmpeg.
```

## Этап 2 — Backend API (3–4 дня)

**Цель:** REST API для приёма задач и выдачи статуса.

**Задачи:**
- [ ] Инициализировать FastAPI проект
- [ ] PostgreSQL: таблицы `jobs`, `users` (Alembic миграции)
- [ ] `POST /jobs` — принять файлы, создать Job, вернуть `job_id`
- [ ] `GET /jobs/{job_id}` — статус + ссылка на видео
- [ ] Celery + Redis: задача `render_job(job_id)` запускает `render_video.py`
- [ ] Автоудаление файлов через 24ч (cron или Celery beat)
- [ ] Валидация: MIME-тип файлов, лимиты размера
- [ ] Rate limiting: 3 задачи/час для анонима

**Команда для Claude Code:**
```
Создай FastAPI backend с эндпоинтами POST /jobs и GET /jobs/{job_id}.
Используй PostgreSQL (SQLAlchemy + Alembic), Celery + Redis для очереди.
Job хранит: id, status (pending/processing/done/error), output_path, expires_at.
```

## Этап 3 — Веб-интерфейс (2–3 дня)

**Цель:** Простая форма на сайте, которой пользователь может пользоваться.

**Задачи:**
- [ ] Страница с формой: drag-and-drop фото, загрузка MP3, поле текста, выбор стиля
- [ ] Прогресс-бар рендеринга (polling GET /jobs/{id} каждые 5 сек)
- [ ] Страница превью: видеоплеер + кнопка скачать
- [ ] Мобильная адаптация (телефон — основное устройство ЦА)
- [ ] Анимация загрузки («Создаём ваше видео...»)

**Команда для Claude Code:**
```
Создай HTML/JS страницу для Поздравлятора: форма загрузки фото (до 30),
MP3, текст, выбор стиля. После отправки — прогресс-бар с polling к API.
После готовности — превью видео и кнопка скачать.
```

## Этап 4 — Telegram-бот (2–3 дня)

**Цель:** Полноценный бот с основным флоу.

**Задачи:**
- [ ] Инициализировать python-telegram-bot v20 (async)
- [ ] Флоу: старт → альбом фото → MP3 → текст → стиль → очередь → уведомление
- [ ] ConversationHandler для пошагового диалога
- [ ] Отправка видео (< 50MB) прямо в чат, крупнее — ссылка на скачивание
- [ ] Уведомление «ваше видео готово» при async рендере

**Команда для Claude Code:**
```
Создай Telegram-бот на python-telegram-bot v20 для Поздравлятора.
ConversationHandler: шаги — фото (альбом) → MP3 → текст → стиль.
После создания Job → ожидание → уведомление с видео или ссылкой.
```

## Этап 5 — Монетизация (2 дня)

**Цель:** Принимать оплату через Telegram Stars.

**Задачи:**
- [ ] Интеграция Telegram Stars Payments API в бот
- [ ] `POST /jobs/{id}/pay` — проверить оплату, убрать водяной знак, пересобрать видео
- [ ] Кнопка оплаты на сайте (через инвойс Telegram)
- [ ] Таблица `payments` в БД

**Команда для Claude Code:**
```
Добавь оплату через Telegram Stars в бот и на сайт.
При успешной оплате: убрать водяной знак, пересобрать видео (render_video.py watermark=False),
обновить Job.is_paid=True, уведомить пользователя новой ссылкой.
```

---

## Этап 6 — Полировка и тесты (2–3 дня)

**Цель:** Готово к публичному запуску.

**Задачи:**
- [ ] Обработка ошибок: что если FFmpeg упал? Уведомить + дать повторить
- [ ] Конвертация HEIC → JPEG (iPhone-фото через `pillow-heif`)
- [ ] Лимиты: красивые сообщения «файл слишком большой»
- [ ] Логирование ошибок (Sentry или простой logging)
- [ ] Landing page (можно взять за основу `kalkulyator-prizvaniya-landing.html`)
- [ ] Тест всего флоу руками: 10 разных видео

---

## Этап 7 — Деплой на VPS (1–2 дня)

**Цель:** Сервис доступен по домену.

**Задачи:**
- [ ] Выбрать VPS: Hetzner CX31 (4 vCPU, 8GB, ~20$/мес) или Timeweb
- [ ] Docker Compose: `api`, `worker`, `redis`, `postgres`, `nginx`
- [ ] Cloudflare DNS + SSL сертификат (Let's Encrypt)
- [ ] Настроить домен (например: `pozdravlyator.ru`)
- [ ] Переменные окружения: `.env` файл
- [ ] Настроить Telegram Webhook

**Команда для Claude Code:**
```
Создай docker-compose.yml для Поздравлятора: сервисы api (FastAPI),
worker (Celery), redis, postgres, nginx с SSL через certbot.
```
